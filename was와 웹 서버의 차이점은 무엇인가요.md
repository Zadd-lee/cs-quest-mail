# 답

[[WAS]]

정리하자면 웹 서버는 정적 컨텐츠 제공에 특화되어있으며, was는 동적 컨텐츠 생성과 데이터 처리에 특화되어 있음

# 꼬리질문 - WAS도 정적 컨텐츠를 제공할 수 있는 데 웹 서버가 따로 필요한 이유는?
WAS가 너무 많은 역할을 담당하면 과부하가 될 수 있다
웹 서버를 따로 분리하면 WAS는 중요한 애플리케이션 로직에 집중할 수 있으며 웹 서버는 정적 리소스를 처리하면서 업무 분담이 가능해짐
시스템 리소스를 효율적으로 관리할 수 잇다
정적 컨텐츠가 많이 사용되는 경우, 웹 서버를 증설하고, 애플리케이션 자원을 많이 사용되면 WAS를 증설하면 된다
이외에도 [[로드밸런싱]]을 하거나, [[3. Resource/개발 문서/캐싱]] 및 압축, [[3. Resource/개발 문서/HTTPS]]등 웹서버에서 처리하도록 할 수 있다
