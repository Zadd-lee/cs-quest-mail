# Record를 DTO로 사용하는 이유는?
# 내 답변
1. Record에 대한 설명
2. DTO 개념
3. 장단점?

Record는 java 언어에서 사용하는 객체의 종류 중 하나이다.
Record는 불변 이라는 특징을 가지고 있다

DTO는 layer 간의 데이터 통신에서 사용하는 개념으로 데이터의 무결성을 지켜주기 위해 불변이라는 특징을 갖고 있어야 한다

그렇기 때문에 불변하는 특징인 Record를 DTO로 사용한다


# 답변
Record는 Java 16에서 정식 출시된 특별한 유형의 클래스로 불변성(Immutable) 을 기본으로 함

기존 클래스와 달리 모든 필드가 final 키워드로 선언되며, 객체 생성 후 변경할 수 없다.
필드 선언만으로 자동으로 생성자,getter,equals()등 메서드를 자동으로 생성해주어 보일러 플레이트코드를 줄일 수 있다.
이러한 특성으로 인해 멀티 스레드 환경에서 데이터가 의도치않게 변경되지 않고 안전하게 전달 할 수 있다.

## 꼬리 질문
- Record로 생성한 모든 객체는 DTO인가요?
	- 아니다. Record는 단순 데이터를 캡슐화 하는 역할 DTO외에 값 객체(Value Object)등 다양하게 사용 가능함
	- DTO는 계층간 데이터 전송을 목적으로 하는 반면 VO는 도메인 모델 내에서 특정 값을 표현하는 객체로 사용됨
	- DTO와 VO 두가지 모두 적합하게 사용할 수 있지만, 목적에 따라 사용 방법이 달라진다
# 배운 점

자바에서 클래스와 객체는 다른 의미이다
클래스 = 설계도
객체 = 설계도를 바탕으로 만들어진 인스턴스
