# 질문
https에 대해서 설명해주세요
# 내 답변
보안 기능을 추가한 http
외에는 잘 모르겠다

# 답변
http(Hypertext Transfer Protocol)는 웹에서 클라이언트와 서버 간 통신을 위한 규약
하지만 http는 암호화되지 않은 평문 데이터를 전송하기 위해 제 3자가 정보를 조회할 수 있는 위험이 있다. 이를 위해 https가 등장

HTTPS(Hypertext Transfer Protocol Secure) 는 http에 데이터 암호화가 추가됨
암호화된 데이터를 전송하기 때문에 제 3자가 볼 수 없도록 할 수 있다

## HTTPS는 어떻게 적용할 수 있나요?
인증된 기관(CA) 에게 인증서를 발급받아야 함
CA에 인증서를 요청하면 CA 이름, 서버 공개키, 서버 정보를 활용하여 인증서를 생성하고 이를 개인키로 암호화해 서버로 전송함
이때 인증서는 CA 개인키로 암호화하니 신뢰성을 확보할 수 있다.

## HTTPS 동작 원리에 관해 설명
클라이언트가 서버로 최초 요청할 때 암호화 알고리즘, 프로토콜 버전, 무작위 값을 전달함
이를 받은 서버는 클라이언트에게 암호화 알고리즘, 인증서, 무작위 값을 전달.
클라이언트는 서버의 인증서를 CA 공개키로 복호화하여 검증
검증이 끝난 이후 클라이언트와 서버에서 생성된 무작위 값을 조합하여 Pre Master Secret 값을 생성하여 서버 공개키로 암호화하여 전달함

서버는 전달받은 암호화된 데이터를 개인 키로 복호화하여 Pre Master Secret을 얻음
클라이언트와 서버의 일련의 과정을 통해 Pre Master Secret을 Master Secret으로 변경하고 해당 정보를 이용해 세션 키를 생성

이 과정을 tls 핸드 쉐이크라고 하며 이후부터 클라이언트와 서버는 세션 키를 활용한 대칭키 암호화 방식으로 데이터 송수신을 수행
# 배운 점
질문에서 보이는 개념들은 쪼개서 설명하자
https에 대한 질문은 http와 https에 대한 개념, https의 동작 원리까지 답변할 수 있었다
